# Day 1: RAG 基础概念 — 让 AI 查资料再回答

## 学习目标

- 理解 RAG 是什么、解决什么问题
- 掌握文档分块（Chunking）策略
- 理解 Embedding（文本向量化）原理
- 用 Chroma 完成一次完整的 RAG 流程

## 核心概念

### RAG = Retrieval-Augmented Generation（检索增强生成）

**一句话**：AI 不靠记忆回答，而是先从你的文档中搜索相关内容，再基于搜索结果回答。

| 方案 | 类比 | 优点 | 缺点 |
|------|------|------|------|
| 直接问 AI | 闭卷考试 | 简单 | 可能编造/过时 |
| 微调模型 | 提前背书 | 效果好 | 贵、慢、需要大量数据 |
| **RAG** | **开卷考试** | **便宜、实时、可控** | 检索质量影响回答 |

### JS 类比

```
没有 RAG: fetch("/api/chat", { prompt: "React 18 新特性？" })
          → AI 可能编造答案

有 RAG:   search(docs, "React 18")           // 第 1 步: 搜索文档
          → 找到相关片段
          → fetch("/api/chat", {              // 第 2 步: 带着文档问 AI
              prompt: "根据以下文档回答: ..." + 片段
            })
          → 准确回答（有依据）
```

## RAG 完整流程

```
1. 离线阶段（建索引）
   文档 → 分块(Chunking) → 向量化(Embedding) → 存入向量库

2. 在线阶段（回答问题）
   用户提问 → 向量化 → 在向量库中检索 → 取 top-k 结果
   → 拼接成 prompt → 调 LLM 生成回答
```

## 关键概念

### 1. Chunking（文档分块）

把长文档切成小块。为什么？
- token 限制：不可能把整篇文档塞进 prompt
- 精准度：小块更容易匹配到精确答案

| 策略 | 方法 | 优缺点 |
|------|------|--------|
| 固定字数 | 每 300 字切一块 | 简单但可能切断句子 |
| 按段落 | 用 \n\n 分割 | 保留语义但长度不均 |
| 递归切分 | 先段落→句子→字数 | 最佳实践，LangChain 默认方案 |

overlap（重叠）：相邻块之间重叠 50-100 字，防止关键信息被切断。

### 2. Embedding（文本向量化）

把文本转成一组数字（向量），语义相近的文本向量也相近。

```
"React 组件"  → [0.12, 0.85, -0.33, ...]  ← 很接近！
"Vue 组件"    → [0.15, 0.82, -0.30, ...]  ←
"Python 列表" → [0.78, -0.12, 0.45, ...]  ← 很远

JS 类比: Embedding = 给文本生成"语义指纹"
传统搜索: 关键词匹配（"React" 搜不到 "Vue"）
向量搜索: 语义匹配（"前端框架" 同时搜到 React 和 Vue）
```

### 3. 向量数据库（Chroma）

存储和检索向量的数据库。类比：

| 概念 | 传统数据库 | 向量数据库 |
|------|-----------|-----------|
| 存什么 | 行和列 | 文本 + 向量 |
| 怎么查 | SQL WHERE | 向量相似度 |
| 类比 | MySQL | Chroma / FAISS / Pinecone |
| JS 类比 | MongoDB | Elasticsearch（但基于语义） |

### 4. 相似度计算

| 方法 | 范围 | 含义 |
|------|------|------|
| 余弦相似度 | -1 到 1 | 1=完全相同，0=无关 |
| L2 距离 | 0 到 ∞ | 0=完全相同，越大越远 |
| 内积 | -∞ 到 ∞ | 越大越相似 |

Chroma 默认用 L2 距离，可以配置成余弦相似度。

## 关键要点

1. **RAG 不是万能的** — 检索质量差，回答也差。垃圾进垃圾出。
2. **分块是核心** — 切太大噪音多，切太小丢失上下文。300-500 字是常用范围。
3. **Embedding 是离线的** — 文档入库时一次性向量化，之后查询只需向量化问题。
4. **Chroma 的默认 embedding 免费** — 用 all-MiniLM-L6-v2 本地模型，不调 API。
5. **RAG 的核心价值** — 让 AI 回答有依据，减少幻觉，能处理私有数据。
